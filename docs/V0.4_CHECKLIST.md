# v0.4 Improvement Checklist (Ralph Loop + TDD)

## Ground Rules (mandatory)
- [ ] TDD for every non-trivial logic change:
  - [ ] Write failing tests first
  - [ ] Implement minimal code to pass
  - [ ] Run full suite before commit
  - [ ] No merge if tests fail
- [ ] Work only on `dev/ralph-v0.4-checklist`
- [ ] Keep commits small and scoped to one checklist item
